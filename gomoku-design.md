# 五子棋游戏开发设计方案

## 一、项目概述

### 1.1 项目背景

开发一款精美的 HTML5 网页版五子棋（Gomoku）游戏，提供人人对战功能。参考经典五子棋游戏的美学设计，打造视觉优雅、交互流畅的游戏体验。

### 1.2 核心需求

- **游戏模式**: 双人对战（PVP）- 本地两人在同一设备轮流下棋
- **棋盘**: 15×15 标准五子棋棋盘
- **胜负判定**: 率先连成五子（横、竖、斜）获胜
- **界面**: 精美中国风/现代简约设计

## 三、UI/UX 设计

### 3.1 整体风格

**主题**: 现代简约中国风融合
**配色方案**:

| 元素 | 颜色 |
|------|------|
| 背景 | 渐变暖灰 `#f5f5f0` → `#e8e4dc` |
| 棋盘 | 温润木色 `#deb887` (或深棕 `#8b4513`) |
| 棋盘线 | 檀木色 `#5c4033` |
| 黑子 | 玄铁黑 `#1a1a1a` + 光泽渐变 |
| 白子 | 羊脂白 `#fefefe` + 阴影 |
| 强调色 | 朱砂红 `#c41e3a` (用于高亮) |
| 文字 | 墨色 `#2d2d2d` |

### 3.3 响应式断点

| 屏幕 | 棋盘格子大小 |
|------|-------------|
| Desktop (>1024px) | 40px |
| Tablet (768-1024px) | 32px |
| Mobile (<768px) | 自适应 90vw/15 |

### 3.4 视觉特效

1. **棋子渲染**:
   - 径向渐变模拟3D球体
   - 落子时有缩放弹跳动画 (0.2s ease-out)

2. **高亮连线**:
   - 获胜五子显示红色高亮边框
   - 轻微脉冲动画

3. **光标预览**:
   - 当前可落子位置显示半透明棋子预览
   - 淡入淡出呼吸效果

4. **背景**:
   - 细微纹理或渐变
   - 角落可有装饰性元素（月亮、松枝等中国传统元素）

## 四、功能规格

### 4.1 游戏流程

```
开始游戏
    ↓
黑方先行 (显示提示)
    ↓
玩家点击棋盘交叉点
    ↓
落子动画 + 音效(可选)
    ↓
检查是否获胜
    ↓
是 → 显示胜利信息 + 结束
否 → 切换玩家 → 继续
```

### 4.2 交互规则

| 操作 | 行为 |
|------|------|
| 点击空交叉点 | 放置当前玩家颜色的棋子 |
| 点击已有棋子 | 无效/忽略 |
| 游戏结束后点击 | 无效/显示重开按钮 |

### 4.3 胜负判定算法

检查四个方向：
1. 水平 (← →)
2. 垂直 (↑ ↓)
3. 主对角线 (↖ ↘)
4. 副对角线 (↗ ↙)

任一方向连续5子即获胜。

### 4.4 信息面板内容

- 当前玩家指示（黑方/白方 + 颜色图标）
- 已落子数统计
- 游戏状态（进行中/已结束）
- 重新开始按钮

## 五、详细实现步骤（Vue 3）

### 5.1 步骤一：初始化 Vue 3 项目

```bash
cd gomoku
npm create vite@latest . -- --template vue
npm install
```

### 5.2 步骤二：编写入口文件

- `index.html` - 入口页面
- `src/main.js` - Vue 初始化
- `src/App.vue` - 主组件结构

### 5.3 步骤三：编写棋盘组件 (GameBoard.vue)

- Canvas 棋盘渲染
- 鼠标事件处理
- 棋子绘制逻辑

### 5.4 步骤四：编写游戏逻辑 (composables/gameLogic.js)

| 函数 | 功能 |
|------|------|
| `useGameLogic()` | 组合式函数，返回游戏状态和方法 |
| `initBoard()` | 初始化15×15空棋盘 |
| `placePiece(x, y)` | 落子 |
| `checkWin(x, y)` | 检查胜负 |
| `switchPlayer()` | 切换玩家 |

### 5.5 步骤五：测试与优化

- 检查不同屏幕尺寸显示
- 验证胜负判定正确性
- 优化动画流畅度
