# 五子棋游戏开发设计方案

## 一、项目概述

### 1.1 项目背景

开发一款精美的 HTML5 网页版五子棋（Gomoku）联机对战游戏，支持用户注册、房间对战、倒计时等功能。

### 1.2 核心需求

- **用户系统**: 注册/登录功能，数据存储到数据库
- **游戏模式**: 联机对战（PVP）- 两人通过网络实时对战
- **通信方式**: WebSocket 实时通信
- **棋盘**: 15×15 标准五子棋棋盘
- **先手规则**: 随机指定先手玩家
- **胜负判定**: 率先连成五子（横、竖、斜）或超时判负
- **倒计时**: 思考时间 2 分钟/每次，超时判负

│              username + password              │

## 四、功能规格

### 4.1 用户系统

#### 4.1.1 注册
- 用户名：3-20字符，唯一
- 密码：6-20字符
- 存储到 `database.json`

#### 4.1.2 登录
- 用户名 + 密码验证
- 登录成功后进入首页

### 4.2 房间系统

#### 4.2.1 创建房间
1. 点击「创建房间」
2. 服务器创建房间，返回6位房间号
3. 跳转到等待页面，显示房间号
4. 等待对手加入

#### 4.2.2 加入房间
1. 输入房间号
2. 点击「加入房间」
3. 验证房间存在且未满
4. 加入成功，游戏开始

### 4.3 游戏规则

#### 4.3.1 随机先手
- 游戏开始时随机选择黑方或白方先手

#### 4.3.2 倒计时
- 每次落子思考时间：**2分钟**
- 当前玩家开始倒计时
- 落子完成后，切换玩家，新玩家开始倒计时

#### 4.3.3 超时判负
- 倒计时结束（2分钟）前未落子
- 超时方判负，对方获胜

### 4.4 界面提示

| 场景 | 显示 |
|------|------|
| 己方回合 | 己方倒计时 + 「请落子」 |
| 对方回合 | 对方倒计时 + 「对方思考中...」 |
| 倒计时警告 | 红色闪烁（<30秒） |
| 超时 | 「你超时了！」 |

## 六、WebSocket 协议

### 6.1 消息类型

| 消息 | 方向 | 说明 |
|------|------|------|
| `register` | C→S | 注册 |
| `login` | C→S | 登录 |
| `create_room` | C→S | 创建房间 |
| `join_room` | C→S | 加入房间 |
| `place_piece` | C→S | 落子 |
| `register_ok` | S→C | 注册成功 |
| `login_ok` | S→C | 登录成功 |
| `room_created` | S→C | 房间创建 |
| `room_joined` | S→C | 加入成功 |
| `player_joined` | S→C | 对手加入 |
| `game_start` | S→C | 游戏开始 |
| `piece_placed` | S→C | 落子 |
| `timer_update` | S→C | 倒计时 |
| `player_timeout` | S→C | 超时 |
| `error` | S→C | 错误 |

### 6.2 消息格式

```javascript
// 注册
{ type: 'register', username: 'user1', password: 'pass123' }

// 登录
{ type: 'login', username: 'user1', password: 'pass123' }

// 创建房间
{ type: 'create_room' }

// 加入房间
{ type: 'join_room', roomId: 'ABC123' }

// 落子
{ type: 'place_piece', x: 7, y: 7 }

// 游戏开始
{ type: 'game_start', first: 'black', challenger: 'user1', challenged: 'user2' }

// 倒计时
{ type: 'timer_update', timeRemaining: 120000, currentPlayer: 'black' }
```
